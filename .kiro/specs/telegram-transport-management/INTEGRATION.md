## Shared Database Tables

### drivers Table
**Created by:** Admin Panel (`/admin/drivers`)
**Used by:** Both Admin Panel and Telegram Bot

| Field | Admin Panel Usage | Telegram Bot Usage |
|-------|-------------------|-------------------|
| `driver_id` | Generated/assigned by Fleet Manager | Used for registration |
| `driver_name` | Entered by Fleet Manager | Displayed in bot messages |
| `telegram_id` | Read-only, shows registration status | Populated during `/start` |
| `auth_pin` | Generated by Fleet Manager | Used for authentication |
| `vehicle_id` | Assigned by Fleet Manager | Displayed in status messages |
| `status` | Displayed in real-time dashboard | Updated via `/online`, `/offline` |
| `preferred_language` | Set by Fleet Manager | Used for bot message translation |
| `last_telegram_activity` | Displayed as "Last Seen" | Updated on every bot interaction |

### driver_assignments Table
**Created by:** Admin Panel (`/admin/bookings`)
**Used by:** Both Admin Panel and Telegram Bot

| Field | Admin Panel Usage | Telegram Bot Usage |
|-------|-------------------|-------------------|
| `driver_id` | Selected from available drivers | Determines notification recipient |
| `booking_id` | Links to booking being assigned | Displayed in trip details |
| `status` | Displayed in assignment panel | Updated on accept/reject |
| `response_timestamp` | Shows driver response time | Set when driver responds |
| `telegram_notification_sent` | Indicates notification delivery | Set to TRUE after sending |

## API Endpoint Mapping

### Admin Panel Endpoints (Enhanced)

| Endpoint | Purpose | Telegram Integration |
|----------|---------|---------------------|
| `POST /v1/admin/drivers` | Create driver | Generates auth_pin for Telegram registration |
| `GET /v1/admin/drivers?telegram_registered=true` | List drivers | Filter by Telegram registration status |
| `POST /v1/admin/assignments` | Assign driver | Triggers Telegram notification |
| `POST /v1/admin/telegram/broadcast` | Send broadcast | Queues messages to drivers |
| `GET /v1/admin/telegram/analytics` | View analytics | Aggregates bot usage data |

### Telegram Bot Endpoints (New)

| Endpoint | Purpose | Admin Panel Impact |
|----------|---------|-------------------|
| `POST /v1/telegram/webhook` | Receive bot updates | N/A (internal) |
| `POST /v1/telegram/register` | Register driver | Updates drivers.telegram_id, triggers WebSocket |
| `POST /v1/telegram/status` | Update status | Publishes to Redis, updates Admin Panel in real-time |
| `POST /v1/telegram/assignments/:id/accept` | Accept trip | Updates assignment, notifies Admin Panel via WebSocket |
| `POST /v1/telegram/support` | Create ticket | Creates ticket visible in Admin Panel |

## WebSocket Events

### Events from Backend to Admin Panel

| Event | Trigger | Admin Panel Action |
|-------|---------|-------------------|
| `driver:status:changed` | Driver uses `/online` or `/offline` | Update DriverList status badge |
| `driver:registered` | Driver completes `/start` registration | Update registration badge to ✅ |
| `assignment:response` | Driver accepts/rejects trip | Update DriverAssignmentPanel status |
| `driver:location:updated` | Driver shares location | Update map marker position |
| `driver:emergency` | Driver sends `/emergency` | Show modal alert with sound |
| `driver:support:ticket` | Driver creates support ticket | Add ticket to SupportTicketList |
| `broadcast:status` | Broadcast delivery progress | Update sent_count in BroadcastHistory |
